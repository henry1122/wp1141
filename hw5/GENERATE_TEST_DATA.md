# 生成測試資料指南

## 🚀 快速生成測試帳號和貼文

我已經為您創建了一個 API 來生成測試資料。

### 方法 1: 使用預設設定（推薦）

在瀏覽器中訪問：

```
http://localhost:3000/api/admin/generate-test-data
```

這會生成：
- **10 個測試用戶**
- 每個用戶 **5 篇貼文**
- 總共 **50 篇貼文**

### 方法 2: 自訂數量（使用 POST 請求）

使用 curl 或 Postman 發送 POST 請求：

```bash
curl -X POST http://localhost:3000/api/admin/generate-test-data \
  -H "Content-Type: application/json" \
  -d '{"userCount": 20, "postsPerUser": 10}'
```

這會生成：
- **20 個測試用戶**
- 每個用戶 **10 篇貼文**
- 總共 **200 篇貼文**

---

## 📋 生成的內容

### 測試用戶

- **userID**: `user1_xxxxxx`, `user2_xxxxxx`, 等等
- **Name**: 隨機中文名稱（張三、李四、小明等）
- **Email**: `test1@example.com`, `test2@example.com`, 等等
- **Provider**: `credentials`

### 測試貼文

- 隨機從 30 個預設貼文中選擇
- 內容包括：日常分享、心情、問題、邀請等
- 每個用戶會有多篇不同的貼文

---

## ✅ 使用步驟

1. **確保開發伺服器正在運行**
   ```bash
   npm run dev
   ```

2. **生成測試資料**
   - 訪問：`http://localhost:3000/api/admin/generate-test-data`
   - 或使用 POST 請求自訂數量

3. **查看結果**
   - 返回 JSON 回應，顯示創建的用戶和貼文數量
   - 刷新主頁面，應該能看到所有貼文

4. **在主頁面查看**
   - 訪問：`http://localhost:3000`
   - 應該能看到所有用戶的貼文
   - 可以切換「All」和「Following」標籤

---

## 🔄 重新生成

如果需要重新生成：

1. **清除現有資料**（可選）
   - 可以手動刪除資料庫中的用戶和貼文
   - 或直接重新生成（會跳過已存在的用戶）

2. **再次訪問 API**
   - 已存在的用戶會被跳過
   - 只會創建新的用戶和貼文

## ⚠️ 注意事項

1. **資料庫連接**
   - 確保資料庫連接正常
   - 如果失敗，檢查 `.env.local` 中的 `DATABASE_URL`

2. **生成數量**
   - 建議不要一次生成太多（超過 100 個用戶）
   - 可能會需要一些時間

3. **重複生成**
   - 如果 userID 已存在，會跳過該用戶
   - 但會為已存在的用戶創建新的貼文（如果指定）
